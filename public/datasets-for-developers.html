Developer dataset access

Datasets for diabetic AI agents, without harvesting diabetic data.

OpenDiabetic is building a local-first dataset workflow for AI agents, fine-tunes, LLM evaluation, and research tooling. Raw medical data stays governed. Developers get audited access packs, dataset cards, receipts, and clear use boundaries.

Boundary: OpenDiabetic datasets are for research, education, organization, lifestyle support, and evaluation infrastructure. They are not a license to build diagnosis, medication dosing, or emergency triage systems.

Dataset Doctrine

No harvesting

Private diabetic records are not the business model.

Local-first

NAS-held datasets are inventoried locally before any developer access.

Receipts

Every access pack must record source, license, allowed use, risk, and reviewer.

Opt-in research

User-owned data requires explicit consent, de-identification, and revocation path.

Dataset Families

  • CDC NHANES: public health, nutrition, exam, lab, and questionnaire data.
  • CDC BRFSS: lifestyle, risk behavior, chronic condition, and preventive service survey data.
  • UCI Diabetes 130-US Hospitals: public CC BY 4.0 readmission benchmark with sensitive demographic fields.
  • PhysioNet MIMIC-IV: credentialed, restricted EHR research data.
  • OpenDiabetic local vault-derived examples: private, opt-in only.
  • OpenDiabetic synthetic/de-identified examples: generated only after review.

Developer CLI

Inside the repo, use:

npm run datasets -- list
npm run datasets -- show nhanes
npm run datasets -- stage nhanes /mnt/swarm/opendiabetic-datasets/02_STAGED_PUBLIC_DATASETS
npm run datasets -- audit /mnt/swarm --out /mnt/swarm/opendiabetic-datasets/00_AUDIT_SUMMARIES/full-YYYYMMDD --shard-size 10000

The CLI writes manifests, audit summaries, staging READMEs, and receipts. It does not copy raw medical data by default.

What Developers Receive

  • dataset card
  • source/license notes
  • schema/profile summary
  • allowed and blocked uses
  • privacy/PHI risk notes
  • local path or approved staging location
  • receipt ID
  • Honey/Jelly/Propolis conversion status

Request Rule

No developer gets raw dataset access until source, license, PHI/PII risk, allowed use, blocked use, retention, export rules, and model-training permission are recorded.
